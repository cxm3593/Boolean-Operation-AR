Boolean Operation for modeling in AR/VR environment | CSCI 716

Introduction

In traditional modeling softwares like Maya, users work with a flat screen but when 3D positions are projected into a 2D screen, the sense of depth is lost in the process. Modeling with XR devices provide an advantage by viewing models in a 3d environment. There are existing VR modeling applications so this project aims to build a prototype for 3D modeling in AR environment.

Objective

  • Use boolean operation algorithm for 3d modelling in an AR/VR environment.
  • Target platform is magic leap.
    • Since magic leap is not very stable. A fallback option would be Oculus.
  • The user can observe a 3D model and edit it by grabbing another object and intersecting them together.
  • The intersection will invoke boolean operation algorithm and then modify the target object.
  • Final presentation will have a video demo and a short slide presentation.

Challenges

  • How to modify 3d mesh in Unreal at run time?

    • Epic geometry team developed a plugin to do it, it is called Geometry Processing Plugin. It is a little complicated to use this plugin because we need to compile an unreal editor with the plugin from source code. But the plugin is also very powerful. It provides a complete framework for geometry editing at run time.

  • How to implement boolean operation in 3D?

    Boolean operation is derived from map overlapping algorithm which connected to plane sweep algorithm. Text book Computational Geometry by Mark de Berg covers a 2D version of it. A challenge will be extending it for an extra dimension.

  • How to do it in AR/VR?

    Magic Leap works with unreal 4. It provides a SDK for developing AR applications. But magic leap is not very stable. So a backup plan is to use SteamVR or Oculus VR for Oculus development.
